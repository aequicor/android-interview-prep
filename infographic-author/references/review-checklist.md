# Чек-лист ревью диаграммы / инфографики

Прогоняй перед сдачей. Делится на «рендер не сломается», «схема сильная», «темы и доступность».

## A. Целостность и рендер (обязательно)

- [ ] **Тип под задачу.** Поток — flowchart, обмен — sequence, состояния — state, вложенность —
      структурная, данные — ER/class, сравнение — таблица (а не схема). См. `diagram-types.md`.
- [ ] **Mermaid: директива с начала строки** (без ведущих пробелов). Проверка:
      `grep -nP '^\s+(flowchart|sequence|state|class|graph|erDiagram)' файл` → пусто.
- [ ] **Mermaid: узлы с `(){}:,<` в кавычках** (`A["foo, bar()"]`); подписи короткие.
- [ ] **Экранирование в SVG/коде:** `<`, `>`, `&` внутри текстовых подписей и `<code>` — как
      `&lt; &gt; &amp;`.
- [ ] **SVG-текст влезает в блоки:** ширина блока ≥ `символы × 8 + 2×отступ` (14 px). Подписи не
      налезают на стрелки и друг на друга.
- [ ] **viewBox по контенту:** высота = низ последнего элемента + поле; контент не обрезан, нет
      отрицательных координат, нет огромного пустого поля снизу.
- [ ] **Линии-соединители с `fill="none"`** — иначе чёрная клякса вместо линии.
- [ ] Рендерится без «Syntax error» (Mermaid) и без обрезки (SVG).

## B. Сила схемы (визуальное качество)

- [ ] **Цвет семантичен:** ≤2–3 оттенка, оттенок = категория, серый/`--text-dim` для нейтрального.
      `--danger`/`--warn` только по смыслу. Радужного перебора нет.
- [ ] **Один акцент** на 1–2 ключевых элемента; остальное приглушено.
- [ ] **Сдержанность:** без теней, градиентов, свечения, неона; тонкие границы; болд точечный.
- [ ] **Бюджет сложности:** ≤4–5 узлов, ≤4 блока в ряд, подписи ≤5 слов. Перегруз разбит на серию
      (обзор + подпотоки) с текстом-связкой между схемами.
- [ ] **Цикл не кольцом:** петлю несёт возвратная стрелка или пошаговый блок.
- [ ] **Регистр предложения** в подписях; без гигантских номеров и заголовков поверх холста; без
      декоративных эмодзи в узлах.
- [ ] Прогон против `anti-slop.md` — визуальных ИИ-маркеров не осталось.

## C. Темы и доступность

- [ ] **Обе темы:** цвета в SVG — через переменные сайта (`--accent`, `--text`, `--border`…), не
      литералами. Исключение — фиксированный тёмный текст поверх насыщенной цветной заливки.
- [ ] Мысленная проверка «почти чёрный фон»: каждый текст и линия читаемы в тёмной теме.
- [ ] **Доступность:** у `<svg>` есть `role="img"`, `<title>` и `<desc>` с однофразовым описанием.
- [ ] **Цвет — не единственный признак** категории (есть форма/штриховка/подпись).
- [ ] Подпись к схеме на странице — в `<div class="cap">` (контракт `interview-prep-author`).

## D. Финальная проверка (полезно)

- [ ] Открыть страницу в браузере в **обеих** темах: схема читаема, цвета не сливаются с фоном,
      Mermaid перерисовывается при переключении темы.
- [ ] Свернуть глаза / отойти: видна ли структура без чтения подписей? Если рябит — цвета/узлов
      слишком много, вернуться к вычитанию.
