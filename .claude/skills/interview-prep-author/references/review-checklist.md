# Чек-лист ревью страницы/главы

Прогоняй перед сдачей. Делится на «рендер не сломается» и «контент сильный».

## A. Целостность и рендер (обязательно)

- [ ] Файл начинается с `<!DOCTYPE html>` и заканчивается на `</html>` — без мусорных/нулевых
      байтов в хвосте. Проверка: `python3 -c "d=open('F','rb').read(); print(d.count(0), d.rstrip()[-7:])"`
      должно дать `0 b'</html>'`. Если ripgrep считает файл «binary» — в нём есть NUL-байты, обрежь до `</html>`.
- [ ] `data-page` совпадает с `id` главы в `app.js` и с основой имени файла.
- [ ] Подключены `../assets/style.css`, `../assets/app.js` и 3 CDN-скрипта (highlight, kotlin, mermaid).
- [ ] `#sidebar`, `.breadcrumb`, `#toc` оставлены пустыми (их заполняет `app.js`).
      Нет вручную добавленных шапки / оглавления / prev-next / кнопки «пройдено».
- [ ] В коде внутри `<code>`/`<pre>` все `<`, `>`, `&` экранированы (`&lt; &gt; &amp;`).
      Быстрый поиск утечек: `grep -nP '<(Int|String|List|Map|Flow|T)>' файл`.
- [ ] Каждый блок `<div class="mermaid">` — директива с начала строки (без ведущих пробелов);
      проблемные подписи узлов в кавычках. Проверка: `grep -nP '^\s+(flowchart|sequence|state|class|graph)' файл` → пусто.
- [ ] Видео — это `.ytcard` (а не `<iframe>`), `VIDEO_ID` реальный (из `watch?v=`), `?t=СЕК` и `▶ с M:SS` согласованы.
- [ ] Парность тегов: число `<section` = `</section>`; `.layout`, `#content`, `.page` по одному.
- [ ] Нет посторонних символов (например, случайных CJK): `grep -nP '\p{Han}' файл` → пусто.
- [ ] Подпараграфов нет: внутри `.section` нет `<h3>`/`<h4>` как уровня (оглавление одноуровневое);
      мелкий подпункт — `<p class="subhead">`. Проверка: `grep -nE '<h[34]' файл` → пусто.

## B. Навигация (для новой страницы)

- [ ] В `app.js` добавлена запись в `TOPICS` (и при необходимости группа в `GROUPS`).
- [ ] Счётчики на `index.html` обновлены, если они фиксированы числом.
- [ ] Ссылки между страницами относительные и существуют.

## C. Качество контента (Middle+/Senior)

- [ ] 6–10 параграфов; первый — «зачем на собесе».
- [ ] Объяснение «под капотом» + trade-offs + подводные камни, а не только определения.
- [ ] ≥4 примера Kotlin (компилируемые, идиоматичные), ≥2 диаграммы, ≥2 таблицы.
- [ ] Блок «Частые вопросы»: 5–9 карточек с реалистичными вопросами и развёрнутыми ответами.
- [ ] 4–8 реальных источников (приоритет developer.android.com, kotlinlang.org).
- [ ] Факты сверены (особенно версии API, поведение по версиям Android, внутренности структур данных).

## D. Финальная проверка (опционально, но полезно)

- [ ] Открыть страницу в браузере: рендерятся ли mermaid-схемы (нет «Syntax error»), подсветка кода,
      сайдбар/оглавление/прогресс, и открывается ли видео-карточка по клику на нужном тайм-коде.
- [ ] Для высоких ставок — отдельный проход ревьюером (субагентом) на фактическую корректность.
