# Каталог визуальных ИИ-маркеров

Машинную инфографику опознают по набору устойчивых признаков — как и машинный текст. Это визуальный
аналог `tech-author/references/ai-markers.md`. Пройдите готовую схему по таблицам и почините каждый
маркер: не косметикой, а структурным решением (убрать, перекрасить по смыслу, разнести, разбить).

## Цвет

| ✗ Маркер | Почему плохо | ✓ Как надо |
| --- | --- | --- |
| Радужный перебор: шаг 1 синий, 2 жёлтый, 3 красный… | Цвет кодирует порядок, а порядок и так задан стрелками — это чистый шум | Один оттенок на категорию; нейтральные шаги — серые. Цвет тратится только там, где различает смысл |
| 5–7 ярких оттенков на одной схеме | Глазу не за что зацепиться, фокуса нет | 2–3 оттенка: серый + один-два акцента. Один акцент — на 1–2 ключевых узла |
| `--danger`/красный и `--warn`/жёлтый «для красоты» | Эти цвета означают ошибку и предупреждение; ложная семантика сбивает | Красный/янтарный — только для реального провала/риска. Категории — нейтральными акцентами |
| Цвет — единственный признак категории | Теряется при дальтонизме и в ч/б | К цвету добавьте форму, штриховку, тип линии или подпись |

## Эффекты и «объём»

| ✗ Маркер | Почему плохо | ✓ Как надо |
| --- | --- | --- |
| Тени под блоками (drop shadow) | Фирменный признак генератора; имитация глубины, которой в схеме нет | Плоско: сплошная заливка, тонкая граница. Никаких теней |
| Градиентные заливки узлов | Шум, разный вид в темах, «нейросетевой» вид | Сплошной цвет из палитры. Градиент — лишь как исключение для непрерывной физической величины в иллюстративной схеме |
| Свечение/неон/glow вокруг элементов | Декор без смысла, бьёт по контрасту | Убрать. Акцент задаётся цветом и положением, не сиянием |
| Жирные рамки 2–3 px у всех блоков | Тяжело, грязно | Тонкие линии 0.5–1.5 px. 2 px — максимум и только на единственном выделенном узле |
| Скруглённые «пилюли» везде и крупный radius | Игрушечный вид, расфокус | Умеренное скругление (`rx≈4–8` в SVG, `--radius-sm` в HTML); пилюля — осознанно и редко |

## Раскладка и структура

| ✗ Маркер | Почему плохо | ✓ Как надо |
| --- | --- | --- |
| Цикл нарисован кольцом, узлы по окружности | Спутники налезают на стадии, подписи садятся на дугу | Линейная раскладка + возвратная стрелка, либо пошаговый блок. Кольцо — декор, петлю несёт стрелка |
| Всё впихнуто в один холст (6+ компонентов) | Налезающие блоки, стрелки сквозь текст | Декомпозиция: обзор + по схеме на подпоток, текст-связка между ними |
| Подпись лежит на линии стрелки | Нечитаемо, выглядит как ошибка рендера | Подпись — в свободном месте; чаще она не нужна, если ясно «откуда → куда» |
| Налезающие блоки и пересекающиеся стрелки | Схема читается как сломанная независимо от содержания | Проверить рамки на пересечение; стрелку вести в обход Г-образным путём |
| Процесс нарисован деревом, дерево — процессом | Тип не соответствует смыслу | Тип по глаголу запроса (`diagram-types.md`): поток — стрелки, иерархия — вложенность |

## Текст и типографика

| ✗ Маркер | Почему плохо | ✓ Как надо |
| --- | --- | --- |
| Заголовки Каждое Слово С Большой / КАПС | Машинный признак, тяжелее читается | Регистр предложения везде, включая подписи в SVG |
| Гигантские номера шагов, огромные заголовки поверх схемы | Декор крадёт место и фокус | Номер мелко внутри/рядом с блоком; заголовок схемы — в `.cap`, не в холсте |
| Абзац текста внутри блока | Блок — это метка, а не контейнер прозы | В блоке ≤5 слов; объяснение — в текст под схемой |
| 4–5 кеглей в одной схеме | Нет ритма, рябит | Две величины: 14 px названия, 12 px подписи |
| Декоративные эмодзи/иконки в каждом узле | Украшательство без смысла | Текст-метка. Иконка — только если несёт смысл и одна на роль |

## Подписи и «вода» в визуале

| ✗ Маркер | Почему плохо | ✓ Как надо |
| --- | --- | --- |
| Легенда на 7 пунктов под схемой с 2 цветами | Машинная избыточность | Легенда только при ≥3 значимых категориях; иначе подписать прямо на узлах |
| Стрелки во все стороны «один-ко-многим» сеткой | Визуальный спагетти-код | Показать главный поток; частные связи — отдельной схемой подпотока |
| Одинаковые по размеру и форме блоки, ничем не сгруппированы | Нет иерархии, всё «важно одинаково» | Сгруппировать цветом/положением; ключевое — акцентом, остальное приглушить |

## Принцип починки

Удаление штампа без структурного решения — полдела. Убрали радугу — значит, цвет теперь **кодирует
категорию**; сняли тень — блок стал **плоским с тонкой границей**; разогнули кольцо — петлю несёт
**возвратная стрелка**. Самый сильный приём остаётся прежним: **вычитание**. Если после чистки схема
стала разреженнее и читается с одного взгляда — маркеры убраны правильно.
